# Проект: Определение рыночной стоимости автомобилей с пробегом

## 1. Назначение и цель проекта

Разрабатывается приложение для определения рыночной стоимости автомобиля с пробегом на основе его технических характеристик, комплектации и данных о других автомобилях.

**Цель:** построить модель, предсказывающую цену автомобиля.

**Ключевые критерии заказчика:**
- качество предсказания;
- время обучения модели;
- время предсказания модели.

**Целевое требование к качеству:** значение метрики RMSE на тестовой выборке должно быть **меньше 2500**.

## 3. Этапы выполнения

### 3.1. Загрузка и первичный анализ данных
- Загрузить данные из CSV-файла.
- Выполнить общий анализ структуры данных (типы, количество записей, пропуски).
- Выявить и обработать аномалии в столбцах (некорректные значения, выбросы).

### 3.2. Предобработка данных

**Обработка пропусков:**
- Заполнить или удалить пропущенные значения в зависимости от природы признака и доли пропусков.

**Обработка аномалий:**
- Анализ и фильтрация некорректных значений (например, `RegistrationYear` в будущем или слишком далёком прошлом).
- Удаление или корректировка выбросов в числовых признаках (`Power`, `Price` и др.).

**Удаление неинформативных признаков:**
- Исключить признаки, которые не влияют на целевую переменную или не несут полезной информации (например, `NumberOfPictures`, идентификаторы, даты, которые не релевантны для предсказания).

### 3.3. Подготовка выборок для обучения
- Разделить данные на обучающую, валидационную и тестовую выборки.
- Выполнить кодирование категориальных признаков (One-Hot Encoding, Label Encoding или порядковое кодирование).
- При необходимости выполнить масштабирование числовых признаков.
- Убедиться в отсутствии утечек данных из тестовой выборки.

### 3.4. Обучение моделей

**Требования к моделям:**
- Обязательно использовать **LightGBM** (градиентный бустинг).
- Минимум одна модель — **не бустинг** (например, линейная регрессия, случайный лес, KNN, Ridge, Lasso, Decision Tree).
- Для каждой модели провести подбор как минимум двух-трёх гиперпараметров (для LightGBM — ограниченное число комбинаций ввиду длительности обучения).

**Контролируемые параметры:**
- Качество предсказания (RMSE).
- Время обучения модели.
- Время предсказания модели.

### 3.5. Сравнительный анализ моделей

Построить сводную таблицу со следующими характеристиками для каждой модели:

| Модель | Гиперпараметры | RMSE на валидации | Время обучения | Время предсказания |
|--------|----------------|-------------------|----------------|---------------------|

**Анализ:**
- Какая модель показала лучшее качество?
- Какая модель оптимальна по сочетанию «качество — скорость»?
- Учитываются ли критерии заказчика (качество, время обучения, время предсказания).

### 3.6. Выбор лучшей модели и финальная оценка
- Выбрать модель, наилучшим образом удовлетворяющую критериям заказчика.
- Оценить её качество на **тестовой выборке**.
- Проверить выполнение требования: **RMSE < 2500**.

### 3.7. Выводы
- Указать лучшую модель и её гиперпараметры.
- Привести значение RMSE на тестовой выборке.
- Сделать заключение о пригодности модели для использования в приложении.
